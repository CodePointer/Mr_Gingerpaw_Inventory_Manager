# 前端UI设计

前端UI在完成登录之后，主要分为4个标签页：主页，物品，草稿，我的。

## 主页

主要功能：展示常用的一些信息性的卡片。包括：

- 智能输入卡片：使用文字提交相应的内容。放置一个文本框在这里，下面有“发送”按键，发送之后会提交至草稿部分，生成一个"AI-<时间戳>"的草稿单，通过与后端通信完成草稿的生成。加入 自然语言提示，帮助用户理解如何通过文本框生成草稿。例如，类似 ChatGPT 的“我需要添加5包大米到厨房”的输入。
- 待检查的物品。整理成一个卡片组件，每个物品左侧有一个选择按钮，总体下方是提交按钮。每次可以显示4-8个。

## 物品

主要功能：展示物品列表，完成物品的修改、编辑等操作。

- 创建与编辑按钮：完成物品的创建、以及批量编辑。按下批量编辑按钮后，下方物品出现多选框，同时上方按钮变为“编辑x项” “取消”两个按键。进一步选择编辑后，进入batch的编辑方式。
- 搜索框：按照物品名称进行搜索。
- location筛选框：单选。可以不选，即全部location。
- tag筛选框：多选。同时提供创建新tag的方式，应该是一个加号，点击后用户可以输入名称，创建新的tag。输入名称为空，则不创建新的tag。
- 物品名称的卡片：展示物品的名称、单位、位置、标签等信息。标签右侧应该是物品的数量。还有一个部分可以通过输入框以及上下按钮调整物品的数量，并展示在卡片中。同时，还有一个编辑按键，可以修改单个物品。卡片用颜色表示物品增加or减少了。深灰色是删除的物品，红色是减少的，绿色是增加的。
- 底部为翻页功能，可以设置每页的数量，显示一共多少页，并且可以输入第x页进行跳转。

物品信息编辑应该是一个跳出的窗口进行操作。


## 草稿

主要功能：缓存用户操作，用户确认之后才提交至服务器。

- 组织方式类似于订单：置顶的是人工修改的订单。而其他则是带有时间戳的AI订单。
- 如果AI订单发送到了后端，但还没有返回回来信息，则显示“处理中”。
- 单个订单中，不存在重复的物品。重复的物品增减应该进行合并。
- 每个订单中，可以有一个按钮来选择删除单条记录。同时每个订单会有全部提交和全部取消两个选项。


## 我的

主要功能：展示用户的信息。

- 顶部为用户的信息，包括用户名、邮箱等。有相应的按钮可以进行修改。
- 家庭相关卡片：用户可以在此处切换自己当前查看的家庭。同时有一个按钮为创建家庭。用户还可以在此处生成家庭邀请码，以及输入邀请码加入家庭的功能。
- 下方按钮为修改密码、密保问题。
- 底部为退出登录。


# 🗂️ 前端项目代码优化总结

## 📁 1. 项目结构优化

当前目录结构较为完整，但在模块划分和组件复用性方面可以进一步优化：

### 🌱 项目结构建议

* **Pages:** 将功能页面模块化，并在 `(tabs)` 之外进一步区分主要页面和辅助页面。
* **Components:** 进一步拆分通用组件，例如表单组件、卡片组件、弹窗组件等，提高复用性。
* **Hooks:** 将 `hooks` 目录中的逻辑进一步分类，例如权限相关、数据获取相关等。
* **Styles:** 可以考虑将样式文件进一步拆分，例如 `colors.ts` 可以与 `shared.ts` 合并为 `theme.ts` 或拆分为 `typography.ts`, `layout.ts` 等。

## 🌐 2. API 封装优化

### ✅ API 调用封装

* 建议将 API 调用函数从 `utils/api.ts` 中进一步拆分，例如 `authApi.ts`, `familyApi.ts`, `itemApi.ts` 等，以减少模块耦合。
* 可以为每个模块创建一个 `service` 层，方便后期扩展和调试。

### 🌐 Token 处理

* 目前已经在 `AuthProvider` 中实现了 token 管理，但可以进一步引入自动刷新机制，避免 token 过期导致的登录状态丢失。

---

## 📦 3. 前端状态管理

当前主要依赖 Context API 进行状态管理。对于家庭库存管理这种层级结构复杂的应用，建议引入更强大的状态管理工具，例如：

* **Redux Toolkit**（推荐）：可以在复杂状态管理需求出现时逐步迁移，特别是需要跨组件共享数据时。
* **Zustand**：更轻量级的替代方案，特别适合中小型项目。

---

## 🧩 4. 组件划分与复用

目前的组件划分可以更细化，例如将 `ItemModal.tsx` 拆分为以下几个组件：

* **Modal**（基础弹窗组件）
* **ItemForm**（物品信息表单）
* **TagSelector**（标签选择器）

这不仅提高了组件复用性，还能降低各组件的复杂度。

---

## 🎨 5. UI 交互与体验优化

### ✨ 交互优化

* 例如长按删除、滑动归档等手势交互。

### 🎥 动画与反馈

* 可以考虑使用 `react-native-reanimated` 或 `Framer Motion` 提升用户体验。

---

## 🚀 6. 未来扩展方向

* **国际化支持**：可以考虑在初期就设计好语言包结构，后期支持多语言扩展更方便。
* **数据缓存与离线处理**：考虑到移动应用的网络状况，后期可以引入 `redux-persist` 或 `react-query` 实现数据缓存。
